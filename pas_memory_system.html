核心问题：为什么记忆系统如此关键？

🤯 上下文窗口有限

LLM 每次对话只有有限的上下文窗口（4K~128K tokens），无法记住跨会话信息。记忆系统是 LLM 突破短期记忆限制的唯一解法。

🔒 隐私数据不出盒

用户的个人记忆（日程、习惯、偏好、关系网络）是最隐私的数据。必须在本地盒子优先存储，云端只存加密摘要。

📱 多设备无缝接续

用户在手机开始的对话，在盒子 Web 端继续，在 PC 上结束。三端记忆必须实时同步，做到「无缝接续」体验。

现有技术分析：各存储技术的适用性评估

🔍 向量数据库 主流方案

将文本转为嵌入向量存储，支持语义相似度检索。是目前 RAG 架构的标准组件，成熟度高。

语义检索 ✅ 近似匹配 ✅ 精确查询 ❌ 关系推理 ❌
代表：ChromaDB（本地嵌入）· Qdrant（高性能）· Milvus（企业级）

🕸️ 图数据库 新兴方向

以图结构建模实体关系，适合人物关系网络、知识图谱。GraphRAG 是目前最热的研究方向之一。

关系推理 ✅ 路径查询 ✅ 语义检索 ⚠️ 嵌入成本高 ⚠️
代表：Neo4j（经典）· Kuzu（嵌入式）· FalkorDB（Redis协议）

🗝️ KV + 关系数据库 基础底座

KV 存储用户偏好配置（毫秒级读取），关系型 DB 存对话历史结构化数据，形成精确查询底座。

精确查询 ✅ 事务保障 ✅ 语义检索 ❌ 关系推理 ⚠️
代表：Redis（KV缓存）· SQLite（端侧轻量）· PostgreSQL（服务侧）

🧠 记忆框架 直接可用

专为 AI 记忆设计的框架层，抽象了存储细节，提供记忆提取、更新、遗忘等 AI-native 接口。

开箱即用 ✅ AI原生 ✅ 可定制 ⚠️ 私有部署 ✅
代表：Mem0（★强烈推荐）· Zep（对话记忆）· Memoripy（轻量）

📑 全文检索引擎 辅助组件

关键词精确检索，与向量检索互补形成混合检索（Hybrid Search），提升召回率和准确率。

关键词检索 ✅ BM25 ✅ 语义检索 ❌ 轻量嵌入 ✅
代表：Elasticsearch· MeiliSearch（Rust轻量）· Tantivy（嵌入式）

📦 对象存储 盒子专属

存储多模态记忆：图片、音频、视频片段。盒子侧用 MinIO，云端用 S3/COS，统一接口访问。

多模态 ✅ 大文件 ✅ 语义检索 ❌ S3兼容 ✅
代表：MinIO（本地）· 腾讯COS（云端）· Cloudflare R2（无流量费）

技术选型落地：各层推荐组合
📱 手机端 (端侧)
轻量优先
  • SQLite 本地会话缓存，最近 50 条；无需网络依赖
  • KeyChain/SecureStorage 用户偏好、Token、设备密钥安全存储
  • IndexedDB Web 端会话草稿 & 离线缓存
  • 内存 Context Window 当前会话 Working Memory，随进程生命周期
📦 盒子侧 (核心)
全量存储
  • Mem0 (self-hosted) 记忆框架核心，统一 API，内置遗忘曲线
  • ChromaDB 本地向量存储，bge-m3 嵌入，语义检索
  • PostgreSQL 对话历史、实体关系表、结构化元数据
  • Redis 活跃会话缓存、热点记忆、Session KV
  • MinIO 图片、音频、文档对象存储（S3兼容接口）
☁️ 云端 (协同)
摘要备份
  • Qdrant Cloud 加密摘要向量存储，跨盒子记忆同步
  • Cloud PG (CDB) 用户账号、设备管理、跨设备元数据
  • 腾讯 COS / R2 加密多媒体备份（可选，用户自主控制）
  • KMS (密钥管理) 用户持有加密密钥，厂商无法解密原文
7
同步策略与冲突解决

🔄 同步策略设计

⚡
实时同步（<100ms）：活跃会话的 Working Memory，通过 WebSocket 在盒子和手机间实时同步，用户感知无缝
⏱️
批量同步（每 5 分钟）：新产生的记忆向量和结构化事实，从手机增量推送至盒子，盒子处理后写入存储
🌙
夜间同步（每天 02:00）：盒子侧将加密记忆摘要异步上传至云端，用于多盒子备份和异地恢复
📴
离线降级：盒子不可达时，手机使用本地 SQLite 缓存运行，重连后自动增量合并，不丢失数据

⚠️ 冲突解决规则

🏆
盒子侧优先（Source of Truth）：盒子是主记忆节点，所有设备以盒子版本为权威，手机数据视为「待同步」状态
🕐
时间戳 + Vector Clock：每条记忆携带逻辑时钟，同内容不同版本取最新时间戳，并保留旧版本快照供回溯
🔀
语义冲突检测：Mem0 检测到同一实体两条矛盾记忆时（如「用户喜欢咖啡」vs「用户不喝咖啡」），保留最新记录并标注「已更新」
🗑️
遗忘曲线：低重要性记忆按 Ebbinghaus 遗忘曲线降权，90天未访问的记忆自动归档压缩至冷存储
8
隐私安全设计：数据主权在用户手中
🔐

隐私原则：原始记忆永不离开盒子

盒子本地存储全量原文 云端只存 AES-256 加密摘要 用户持有加密密钥（厂商无法解密） 支持完全断网私有化部署 用户可一键抹除所有记忆 记忆访问日志可审计 传输层 TLS 1.3 全程加密 支持导出 JSON 数据迁移

🔑 用户密钥控制

用户注册时生成唯一加密密钥对，存储在盒子 TEE 或手机 Secure Enclave，厂商服务器零知识。

👁️ 记忆可见性控制

用户可以为每条记忆设置私密/共享/公开等级，共享记忆可选择性开放给家庭成员设备。

🚫 记忆删除权

支持删除单条记忆、删除某时间段记忆、「忘记我说的一切」全量删除，并级联清除云端备份。

📊 访问审计日志

所有记忆读写操作记录访问日志，用户可在 Web UI 查看「谁在什么时候读取了什么记忆」。
