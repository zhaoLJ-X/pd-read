先搞清楚：OpenClaw 原生记忆 vs MemOS 各有什么能力？

🦞 OpenClaw 原生记忆

龙虾框架原生内置，以 Markdown 文件为存储基础，带双层架构 + SQLite向量索引的混合检索系统。开箱即用，对盒子非常友好。

  • ✅双层架构：每日日志 YYYY-MM-DD.md + 长期记忆 MEMORY.md
  • ✅混合检索：BM25 关键词 + 向量语义，权重可调
  • ✅本地向量：SQLite + sqlite-vec，单文件零依赖
  • ✅自动刷新：上下文压缩前自动写入记忆
  • ✅本地 Embedding：支持 GGUF 模型，不依赖云 API
  • ✅可读可编辑：Markdown 文件人类可直接审阅/修改
⚠️无原生多端同步机制
⚠️无记忆重要性评分 / 自动遗忘
⚠️无实体抽取 / 知识图谱能力

🧬 MemOS 2.0（星尘）

将记忆视为 第一类系统资源，提供类操作系统的调度层。原生支持 OpenClaw 插件集成，两种部署模式（云端/本地）。

  • ✅统一记忆 API：add / search / edit / delete 四类操作
  • ✅结构化记忆图谱：可检查、可编辑，非黑盒存储
  • ✅MemCube 隔离：多用户/多项目记忆隔离 + 受控共享
  • ✅异步调度：Redis Streams 驱动，毫秒级延迟
  • ✅多模态记忆：文本、图片、工具痕迹原生支持
  • ✅自然语言纠错：支持用自然语言反馈修正记忆
  • ✅MCP 协议：Model Context Protocol 原生支持
⚠️完整部署依赖 Redis，盒子资源需规划
⚠️相比原生 OpenClaw 记忆，配置复杂度更高

⚡ 两者协同 = PAS 最优解

OpenClaw 原生记忆提供轻量、透明、可读的本地存储基础；MemOS 提供智能调度、图谱构建、多端同步的高级能力。叠加后形成互补。

  • 🔵OpenClaw 负责：实时会话上下文管理、文件写入、本地检索
  • 🟣MemOS 负责：记忆结构化、重要性调度、跨端同步、图谱演化
  • 🟢PAS 自研：云端加密同步层、隐私策略执行、手机端轻缓存

设计原则：四条不可违背的约束
🔒
原文永不出盒
记忆原文只存盒子侧。云端 & 手机端只存加密摘要或临时缓存，用户持有唯一密钥。
📦
盒子是 Source of Truth
所有设备以盒子记忆版本为权威。手机、Web 端的记忆均视为「待同步」副本。
⚡
检索不阻塞推理
记忆写入异步执行，读取走 Redis 热缓存，保证对话响应延迟不受记忆操作影响。
🧩
框架层分离
OpenClaw 原生记忆处理实时上下文；MemOS 处理结构化治理。职责分离，互不干扰。

各端职责详解：每层存什么、用什么、为什么
📱
手机端
轻量 · 离线可用
💬 会话缓存
OpenClaw
SQLite 存最近 50 条对话。盒子不可达时支持离线推理。重连后增量同步。
📄 MEMORY.md 快照
OpenClaw
从盒子定期拉取的记忆摘要 Markdown，直接注入对话 System Prompt，实现「记住你」的感知。
⚡ MemCube 热缓存
MemOS
MemOS 判定为 Hot 的高频访问记忆 Cube，主动推送至手机端本地，检索不走网络。
🔑 加密密钥
PAS 自研
存入系统级安全存储（iOS KeyChain / Android Keystore）。密钥用于解密云端同步的记忆摘要。
📦
盒子侧
全量 · 权威 · 本地
📝 每日日志 / 长期记忆
OpenClaw
OpenClaw 原生的 Markdown 文件双层结构。人类可读、可版本管理，是记忆的「原文」所在地。
🔍 混合向量检索
OpenClaw
SQLite-vec 向量 + FTS5 BM25 混合检索，本地 Embedding（bge-m3 / GGUF），NPU 加速，零云依赖。
🧬 MemOS 记忆图谱
MemOS
MemOS 在 OpenClaw 原始记忆之上构建结构化图谱，抽取实体关系，支持图谱级检索和演化。
📊 重要性分层
MemOS
MemOS Scheduler 按 Hot/Warm/Cold 分层调度，高频记忆常驻 Redis，冷记忆压缩归档。
🖼️ 多模态记忆
MemOS
MemOS 原生支持图片、工具调用痕迹的记忆存储（MinIO），配合 CLIP 向量化实现图片语义检索。
☁️
云端
备份 · 同步 · 零知识
🔐 加密摘要备份
MemOS
MemOS Cloud Sync 将盒子记忆摘要（非原文）AES-256 加密后上传。用于盒子损坏时的恢复，以及多盒子场景的记忆迁移。
📄 MEMORY.md 加密备份
OpenClaw
OpenClaw 精炼后的长期记忆文件，加密上传云端作为灾难恢复底线。手机端可凭密钥解密拉取。
👤 账号 & 设备管理
PAS 自研
记录用户账号、绑定设备列表、MemCube 分配关系，不含任何记忆内容。
🌍 公共知识 RAG
PAS 自研
面向所有用户的公共知识库，不包含个人信息。与个人记忆完全分离。

7
三端同步策略

📱 手机 ↔ 盒子

⚡
实时（<100ms）：WebSocket 长连接，当前会话 Working Memory 实时双向同步
📤
批量推送（5分钟）：手机新产生的记忆（日志片段）增量推送盒子，由 MemOS 消化处理
📥
拉取 MEMORY.md（每天）：手机定期从盒子拉取精炼后的长期记忆快照
📴
离线降级：盒子不可达时，手机使用本地 SQLite 缓存 + MEMORY.md 快照继续工作，重连后自动增量合并

📦 盒子 ↔ 云端

🌙
夜间批同步（02:00）：MemOS 将记忆摘要加密后上传云端，做多盒子备份
📄
MEMORY.md 备份（每天）：OpenClaw 精炼后的长期记忆文件加密上传
🔐
零知识加密：用户密钥不上传，云端存储内容厂商无法解密
🔄
灾难恢复：盒子换新时，凭账号 + 密钥从云端下载加密备份，本地解密还原全部记忆

⚠️ 冲突解决

🏆
盒子优先（Source of Truth）：所有端以盒子版本为最终权威
🕐
逻辑时钟：每条记忆携带 Vector Clock，取最新版本，旧版本保留快照
🧬
MemOS 语义冲突检测：相同实体矛盾记忆取最新，标记「已更新」，保留历史
👁️
用户自然语言纠错：MemOS 支持「不对，我不喜欢咖啡」直接修正记忆

隐私设计：数据主权完全在用户手中
🔐
核心隐私原则：Markdown 原文永不离开盒子，云端零知识
盒子本地存储所有原文（OpenClaw Markdown） 云端只存 AES-256 加密摘要（MemOS Cloud） 用户独享加密密钥，存入 TEE / Secure Enclave 厂商服务器无法解密任何内容（零知识） 支持完全断网私有化部署（盒子 + 手机局域网） 用户可一键 export 全部 Markdown 文件 MemOS 记忆删除 → 级联清除云端备份 记忆访问日志可审计（OpenClaw 会话记录）
